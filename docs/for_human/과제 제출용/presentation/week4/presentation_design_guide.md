## 3. 슬라이드별 디자인 권장사항

### Slide 1: 타이틀
- 중앙 정렬 대형 로고/타이틀
- 배경에 은은한 게임 UI 요소 (체력바, 레벨 아이콘 등) 일러스트
- 팀명과 발표자 이름은 하단 소형 텍스트

### Slide 2: 기획 배경
- 왼쪽: 지루한 To-Do 리스트 이미지 (회색톤, 흐릿하게)
- 오른쪽: 게이미피케이션 캘린더 이미지 (생생한 컬러)
- **Before → After** 대비 구조
- 질문 텍스트를 큰 따옴표와 함께 배치

### Slide 3: 프로젝트 개요
- 중앙에 핵심 가치 한 줄 ("일상을 성장시키는 롤플레잉 캘린더")
- 하단에 3개 키워드 카드: `📅 캘린더` · `🎮 게이미피케이션` · `🤖 AI 페르소나`
- Google OAuth 로그인 아이콘 + 간략 설명

### Slide 4: 핵심 기능 1 — 게이미피케이션
- 좌측: 보상 흐름도 (일정 완료 → EXP 획득 → 레벨업 → 칭호)
- 우측: 페널티 흐름도 (기한 초과 → 칭호 강등 → 부정적 수식어)
- 하단: 일일 퀘스트 예시 카드 3개 나열
- 골드(#FFD700)로 보상, 레드(#FF6B6B)로 페널티 색상 구분

### Slide 5: 핵심 기능 2 — LLM & 잔디 쇼케이스
- 상단: 잔디 그래프(GitHub Contribution Graph) 목업 이미지
  - 시안(#06D6A0) 컬러 그라데이션 활용
- 하단: LLM 페르소나 대화 변환 예시
  - 입력 → AI 변환 → 출력 플로우를 말풍선 형태로

### Slide 6: 핵심 기능 3 — 이탈 방지 & 알림
- 타임라인 형태: `활동 중 → 13일차 경고 알림 → 14일 휴면 → 복귀 보상`
- 복귀 보상 패키지를 게임 아이템 박스 이미지로 시각화
- Push 알림 아이콘과 함께 알림 기능도 간략 표시

### Slide 7: 기술 스택 및 아키텍처

## 5. 일반 디자인 원칙

1. **한 슬라이드 = 하나의 메시지** — 텍스트를 최소화하고 키워드·다이어그램 위주로
2. **6-6 규칙** — 한 슬라이드에 6줄 이내, 한 줄에 6단어 이내
3. **여백 활용** — 슬라이드 가장자리 최소 10% 마진 확보
4. **애니메이션은 절제** — 페이드 인(Fade In) 정도만 사용, 과도한 전환 효과 지양
5. **일관성 유지** — 모든 슬라이드에서 동일한 헤더 위치, 폰트 크기, 컬러 규칙 적용
